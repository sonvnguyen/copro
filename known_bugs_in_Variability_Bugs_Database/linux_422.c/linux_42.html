Linux
NULL function pointer dereferenced if SHMEM is enabled but not TMPFS
linux/f7ab9b4

The 'readpage' field of `shmem_aops' only points to a concrete function when TMPFS is enabled, otherwise it points to NULL. GPU driver for Intel i915 depends on SHMEM but not on TMPFS, and makes it possible for a NULL readpage() to be dereferenced by read_cache_page().
Related to commit ca9ab10033d190c1ede85fdf456307bdfdabf079

But fixed by commit f7ab9b407b3
