Linux
Undefined reference to macro when !BLK_CGROUP
linux/f48ec1d

Macro CFQ_WEIGHT_DEFAULT is conditionally defined in blk-cgroup.h, only when BLK_CGROUP is enabled. This macro is referenced in block/cfq-iosched.c (which #include blk-cgroup.h) in a context where the presence of BLK_CGROUP is not guaranteed. Thus, if BLK_CGROUP is disabled, the compiler cannot find any definition for CFQ_WEIGHT_DEFAULT.

But fixed by commit f48ec1d7885
